# 对不齐 · Misaligned

> 对不起，你的排版对不齐！

一个为公众号原创推文设计的"一键排版"网页工具。左边写、右边实时预览，写完点一下复制就能直接粘贴进公众号平台后台，正常预览、发布推文。

---

## 1 这是做什么的

写公众号文章时，最让人头疼的不是写内容，而是排版：什么字体、行距多少、图怎么放、标题怎么突出……一篇文章光排版就能耗掉一两个小时。

「对不齐」做的事情是：

- 你只管写内容，把它分成"标题 / 正文 / 图片 / 引用"这种简单的块
- 工具帮你按选好的风格自动排好版（杂志感 / 报纸感 / 便签感 / 你自定义的风格）
- 一键复制，粘贴进公众号后台直接发

## 4 五分钟上手

按"**确定风格 → 排版 → 发布**"三步走。

### 一、确定风格

- **内置模板**：顶部 tab 切换「杂志 / 报纸 / 便签」，每种自带两套预设配色。
- **自定义配色**：点顶部「配色」按钮，自由调主色 / 辅色 / 点缀色，预览实时变化。
- **AI 生成专属模板**：点顶部「＋」→ 复制弹窗里的 Prompt 给 AI（Claude / ChatGPT 都行）→ 把 AI 返回的 JSON 保存为 `.json` → 上传，顶部多出一个 tab 自动切换。

### 二、排版

- **已有 Markdown 文稿**：点编辑区顶部「导入 Markdown 文件」，按 `# / ## / ### / > / ---` 自动解析；封面与结尾会被保留。
- **从零开始写**：编辑区底部「加块」按钮分三组——左边「引言」、中间「标题 / 正文 / 引用 / 分隔线 / 图片」、右边「结尾」。新块插在你正在编辑的块下方。
- **内容微调**：点击直接编辑；拖块左侧 ⋮⋮ 手柄排序；选中文字按 Ctrl+B/U/I 加粗 / 下划线 / 斜体；选中文字点「高亮」做马克笔效果。
- **批量操作**：点块右侧「✓」多选，或在空白处拖出蓝色框选；整组删 / 复制 / 改层级 / 拖动。
- **插入图片**：图片块一次可选最多 9 张（单张 ≤ 3MB），下方可以加图注（不写不会显示）。

### 三、发布

1. 等所有图片块显示「✓ 全部已上传」。
2. 点右上角「复制 HTML」。
3. 公众号后台 → 新建图文 → 粘贴 → 预览 → 发布。

> 上传的图片有有效期，复制 HTML 后请尽快粘贴进公众号发布。微信会把图片转存到自己的服务器，发布之后就不受影响了。

## 5 自定义模板

如果三种内置模板都不喜欢，可以让 AI 给你生成专属模板：

1. 顶部点「＋」按钮
2. 复制弹窗里的 Prompt（有"无参考图"和"有参考图"两个版本）粘贴给 Claude / ChatGPT 等
3. 把 AI 返回的 JSON 保存为 `.json` 文件
4. 上传 JSON，会出现一个新 tab，自动切到这个新模板

可以**一次上传多个**自定义模板，每个都会变成一个独立 tab，和内置模板并列。鼠标 hover tab 上会出现「×」，点掉就删除那个模板（如果删的是当前激活的，会自动回到「杂志」）。

**自定义模板也支持配色面板调色**：AI 生成的 JSON 里会带 `colors` 字段（这套风格的初始三色 `text` / `bg` / `accent`，可选 `decor`），HTML 里相应位置写成 `{text}` `{bg}` `{accent}` `{decor}` 占位符。上传后配色面板的预设按钮显示「自定义」，点它可以**恢复到该模板的初始配色**；调整任意颜色 → 预览实时变化；切到别的模板再切回来，颜色记忆保留。

> 旧版 JSON 用的是 `primary` / `secondary` / `accent2` 字段名（以及对应的 `{primary}` / `{secondary}` / `{accent2}` 占位符），目前仍能正常加载——上传时会自动按 `primary→text、secondary→bg、accent2→decor` 映射。新生成的模板请直接用新字段名。

**H1 自动编号占位符**：每个 H1 会被自动编号，渲染时替换 JSON 里的占位符：

| 占位符 | 渲染示例（第 3 个 H1 时） |
| --- | --- |
| `{number}` | `3` |
| `{numberPad2}` | `03` |
| `{numberEng}` | `THREE` |
| `{numberCn}` | `叁` |

`{numberEng}` 和 `{numberCn}` 支持到 99（`NINETY-NINE` / `玖拾玖`），超出回落到阿拉伯数字。AI 生成 JSON 时建议挑一个跟风格搭的占位符。

> ⚠ 自定义模板只存在浏览器会话里，**刷新或关闭页面后会全部消失**。把 JSON 文件保存好，下次再用重新上传。

## 8 技术说明

- 纯 HTML + CSS + 原生 JS，**零依赖**
- 无打包步骤，源文件即产物
- 用到的外部资源：
  - Google Fonts CDN：`Playfair Display`（英文 logo 字体）
  - 第三方图片托管服务：用于把本地图片转成可被公众号抓取的 URL
- 富文本复制使用 `navigator.clipboard.write()` + `ClipboardItem` 写入 `text/html`，降级方案是 `contenteditable + execCommand('copy')`

### 8.0 公众号兼容性红线（改模板前必读）

公众号编辑器粘贴 HTML 时会做"标签白名单过滤"：

- **会被剥掉的**：`<div>` —— 标签连同上面的内联 `style` **整个被丢弃**，色块、卡片、彩色背景全没了
- **保留的**：`<section>` / `<p>` / `<span>` / `<table>` / `<h2>` / `<h3>` / `<img>` 等

所以三个内置模板（`tmpl1` / `tmpl2` / `tmpl3`）以及通用图片渲染（`renderImage`）**所有色块和容器都用 `<section>` 而不是 `<div>`**。改模板时务必继续遵守。预览侧浏览器对 `<section>` 和 `<div>` 渲染表现完全一致，不会有视觉差异。

> 自定义模板的 AI 生成 Prompt 也已在要求里强调"容器必须用 `<section>`"。如果用户的 JSON 里出现 `<div>` 容器，复制到公众号后会丢样式——这是已知限制，不是 bug。

### 8.1 统一排版规范（所有模板必须遵循）

为了让杂志 / 报纸 / 便签 / 自定义任何模板切换时视觉感受一致，统一一套**字号 / 加粗 / 行高 / 缩进 / 间距**规范。代码里集中定义在 `TYPO` 常量，AI 生成自定义模板的 Prompt 也写死了同一套。

| 块类型 | 字号 | 加粗 | 行高 | 备注 |
| --- | --- | --- | --- | --- |
| h1 | 24px | ✓ | 1.5 | 文章一级标题（用户在公众号输入框写的"主标题"是另一回事） |
| h2 | 18px | ✓ | 1.5 | |
| h3 | 16px | ✓ | 1.5 | |
| body | 16px | ✗ | 1.9 | **首行缩进 2em** |
| quote | 15px | ✗ | 1.85 | italic |
| cover（引言） | 15px | ✗ | 1.85 | italic、淡灰色 |
| footer | 14px | ✗ | 1.8 | |

- **左右内边距**：22px（wrapper 自带或每块自带，最终视觉一致）
- **加粗只用在 h1/h2/h3**，其他块**默认不加粗**；用户可以在编辑器里用「B」按钮或 Ctrl+B 自己手动加粗某段文字
- **首行缩进只对 body**

新增内置模板或修改现有模板时，**字号 / 加粗 / 缩进必须从 `TYPO` 取值**，不要硬编码。

#### 风格特例（允许豁免）

某些块在特定风格下是这套风格的"招牌"，刻意**不遵循 TYPO 规范**。豁免要在代码里加注释解释原因：

| 模板 | 块 | 豁免内容 | 原因 |
| --- | --- | --- | --- |
| 报纸 | cover | 30px / weight 900 / letter-spacing 2px / 不 italic / 通栏深底色 | 模仿真实报纸头版，是这套风格的灵魂 |
| 报纸 | quote | 15px italic + 大引号 + 署名 + 米色双线 | 模仿报纸社论引文 |
| 便签 | quote | 底色用纯 `accent`（不透明）而非半透明叠加 | 让用户调色时引用块底色直接对应点缀色，调色无障碍 |

#### H1 自动编号

每个 H1 块会被自动编号，渲染时三个内置模板和自定义模板都会用到：

| 模板 | 装饰文字 | 编号格式 |
| --- | --- | --- |
| 杂志 | `NUMBER 01` | 两位数 padding |
| 报纸 | `NUMBER · ONE` | 英文单词（1–99） |
| 便签 | `其 壹` | 中文大写（1–99） |

代码里实现在 `numPad2()` / `numEng()` / `numCnUpper()` 三个工具函数。自定义模板可在 JSON 的 h1 里用 `{number}` / `{numberPad2}` / `{numberEng}` / `{numberCn}` 占位符引用对应格式。

### 8.2 图片托管 API Key 的安全性（给开发者看）

> 这一节面向打算 fork 这个项目的人，普通使用者跳过即可。

当前实现里图片托管 API Key 直接写在前端源码里。**对于纯静态部署（如 GitHub Pages），任何前端调用第三方 API 的 Key 都会被浏览器加载到，没有办法靠"环境变量"或"混淆"真正隐藏**——只要打开浏览器调试工具就能看到。

如果你 fork 了这个项目并想把自己的 Key 真正藏起来，标准做法是搭一个**反向代理**。免费方案推荐 [Cloudflare Workers](https://workers.cloudflare.com/)：

1. 在 Cloudflare 注册一个 Worker，把图片托管 Key 写在 Worker 的环境变量里
2. Worker 接收前端 POST 的图片 → 用环境变量里的 Key 调上游 API → 把上游响应转发给前端
3. 前端的 fetch 地址改成你的 Worker URL，源码里完全不出现 Key

这样：
- Key 永不进入浏览器
- 还能在 Worker 层加访问限制（来源域名白名单、速率限制）防止被滥用

需要详细的 Worker 代码示例可以提 issue。

### 8.3 关于"上传后立即删除"

理论上希望"用户在前端删了图，图床那边也立即删除"。但当前使用的免费图片托管服务**没有提供可程序化调用的删除接口**（其响应里的 `delete_url` 是给人手动点的网页），因此暂时只能依赖**自动过期机制**——上传时设置较短的过期时间，到期自动清除。

如果改用其他支持 API 删除的服务（如自建 MinIO、阿里云 OSS、Cloudinary 等），可以实现"删除即删"。

### 8.4 按钮主 / 副样式规范

| 类型 | 视觉 | 用法 | 当前实例 |
| --- | --- | --- | --- |
| 主按钮 | 黑底白字（`.tbtn.primary` / `.tbarbtn.primary`） | "关键操作"或"当前最重要的下一步" | 复制 HTML、使用手册、导入 Markdown、撤销（在有可撤销内容时动态切换） |
| 副按钮 | 白底深字（`.tbtn` / `.tbarbtn`） | 一般操作 | 配色、重做、高亮 / 取消高亮、纸张切换 |

新增按钮时按这套挑 class，不要随便发明第三种颜色。"撤销"是动态例子：`refreshUndoRedoUI()` 在 `historyStack.length > 0` 时给它加 `primary` 类。

### 8.5 图片渲染规则

图片块设计为**通用透明模块**——背景透明、不用"实心色块"挡上层底色，让风格背景（正文白底 / 引言深色 / 结尾深色 / 便签格子等）自然透出。允许通过 `opt.border` / `opt.padding` / `opt.filter` 给图片加**相框装饰**（透明背景的细描边、内边距、滤镜等），相框不会挡底色。

图片块的渲染分两端，逻辑不同：

| 端 | 单图 | 多图 |
| --- | --- | --- |
| **左侧编辑器**（缩略图） | 120×120，`object-fit:cover` 1:1 裁剪 | 120×120 一组 flex 自动换行，相同尺寸 |
| **右侧预览 / 复制 HTML** | 外层 `padding:0 Npx` 收窄居中（N 由 size 决定）；与正文 2em 首字符对齐 | 每张固定宽度 + `inline-block` + `text-align:center`：自动居中、自动换行、保持原比例不裁剪 |

设计取舍：左侧编辑器追求"等大缩略整齐"方便管理；右侧追求"忠于原图"避免裁掉重要内容；同时单图收窄让大图不顶满边距，视觉更干净。改动 `renderImage()` 时这条不要混淆。

#### 大 / 中 / 小三档

每张图独立保存 `size` 字段（`'L' | 'M' | 'S'`，默认 `'L'`）。尺寸定义在 `SIZE_MAP`：

| 尺寸 | 单图（外层 padding） | 多图（每张 cell 宽） |
| --- | --- | --- |
| L（默认） | 32px | 150px |
| M | 80px | 110px |
| S | 110px | 95px |

操作方式：
- **单张**：编辑器侧每张缩略图 hover → 左下角弹出"大 / 中 / 小"小标签，点击切换该图的 `size`
- **整组一键**：图片块下方"全部：大 / 中 / 小"按钮，把所有 image 的 size 同步成那一档（即使有人单独设过也会被覆盖——这是产品需求）

`size` 字段会随撤销重做、复制块一起被记录/复制；自定义模板也会按 size 走对应分支（自定义只需保证 image HTML 模板的 `<img>` 用 `width:100%` 即可）。

#### 上传校验

- 单张 ≤ **3MB**（受 imgBB 免费额度限制；超出就提示用户压缩）
- 仅 JPG / PNG / WebP / GIF / BMP（白名单，其他类型给具体格式名提示）
- 单块 ≤ **9 张**（满了再传会拒绝并提示"先删除几张"）

三类错误都用 `toast()` 给出**可执行的下一步**（不是抽象的"上传失败"）。

### 8.6 区域装饰：cover / footer 不再自带装饰头尾

cover（引言）和 footer（结尾）从"块级装饰"升级为"**区域装饰**"，由 `wrapCoverRegion()` / `wrapFooterRegion()` 负责整段包装。原因是用户经常想在引言/结尾里穿插图片、分隔线、再多写几段——如果装饰跟着每个 cover/footer 块走，就会出现"一加新文字就来一个 — END —"的尴尬。

#### 区域识别（宽松版）

`generateHTML()` 在渲染前扫描 `g.blocks`，得到两个区域边界：

- **引言区**：从开头到**最后一个 cover 块**的索引为止
- **结尾区**：从**第一个 footer 块**的索引到末尾

中间的任何 image / divider / 其他文字块都会被吸进对应区域，作为该区域的内容。比如"结尾文字 → 分隔线 → 又一段结尾文字 → 一张图 → 署名"会被整段套在结尾区的深色底色上。

#### tmpl1/2/3 的职责变化

改动后，cover / footer 模板只负责输出**自身的文字**，不再写装饰头尾：

- 区域装饰头（杂志 — FEATURE —、报纸 Vol. 卷期号 / 副标、便签上分割线等）由 `wrapCoverRegion` 负责
- 区域装饰尾（杂志 — END —、报纸双横线、便签上一根淡线）+ 署名由 `wrapFooterRegion` 负责

修改模板时务必遵守这个分工。如果新增内置模板，模板函数签名要从 `tmpl(blk,c,html,h1Num)` 升级为 `tmpl(blk,c,html,h1Num,opt)`，`opt.region` 取值 `'cover' | 'main' | 'footer'`，模板可以根据 region 调整图注颜色等细节（深色底用浅图注，浅色底用深图注）。

### 8.7 结尾署名（"对不齐"水印）

三个内置模板的 footer 区域底部都要带一行固定署名，作为本工具排版作品的水印：

| 模板 | 署名文案 | 备注 |
| --- | --- | --- |
| 杂志 | `© DuiBuQi_MISALIGNED · TYPESETTING` | 顶部 — END — 仍保留，作为正文与结尾的色块分割补强 |
| 报纸 | `© DuiBuQi_MISALIGNED · TYPESETTING` | 双横线分割之下，accent2 暗红色 |
| 便签 | `—「对不齐」网站排版 —` | 楷体、淡灰、italic |

> "DuiBuQi" 这个拼音必须**有大有小**（`D / B / Q` 大写，其余小写），其余英文部分允许全大写。中式 / 手写感风格用中文署名"—「对不齐」网站排版 —"。

AI 生成自定义模板的 Prompt 也已要求按此规则在 footer 加署名。

### 8.8 Enter / Shift+Enter 约定

contenteditable 在公众号场景下的换行行为容易踩坑。本工具的约定是：

| 块类型 | Enter | Shift+Enter |
| --- | --- | --- |
| h1 / h2 / h3 | 阻止（标题就是单行） | 阻止 |
| body / quote / cover / footer | **失焦确认**（等价于点空白处） | **软换行**（插入 `<br>`，保留同块样式） |

历史 bug：在 footer 等深色区域里直接按 Enter 会让 contenteditable 走浏览器默认行为，新行可能丢掉父级样式继承（如 footer 的浅色文字变成黑色）。改成 Enter 失焦后这个 bug 就不会再触发——想换行请用 Shift+Enter，软换行不会跳出当前块。

### 8.9 破折号约定

为了视觉层次清楚，本工具区分两种破折号：

| 用途 | 符号 | 例子 |
| --- | --- | --- |
| 装饰性小横线 | `—`（单 em-dash） | `— END —`、`— FEATURE —`、便签的 `—「对不齐」网站排版 —` |
| 引用署名前缀 | `——`（双 em-dash） | 报纸 quote 的 `—— 本报特约评论员` |

新增模板时延续这套；AI 生成自定义模板的 Prompt 也已经写明这条。

### 8.10 引用署名（quote 的 byline 字段）

quote 块除 `content` 外还有一个 `byline` 字段（默认空字符串），用于显示"署名"。

- **数据**：保存在 `blk.byline`，所有风格的 quote 块都会保存（切换风格不丢失）
- **编辑器侧**：所有 quote 块下方会渲染一个独立的小输入框（复用图注样式 `.blk-byline`），placeholder 是 "署名（可选，仅报纸风格使用）…"
- **报纸风格**：quote 渲染时若 `byline` 非空，输出 `<section ...>—— xxx</section>`；为空则整行省略
- **杂志 / 便签 / 其他风格**：当前不显示 byline，但数据保留——切回报纸又能看到
- **自定义模板**：quote 模板里**可以**写 `{byline}` 占位符，`tmplCustom` 会按"非空 → `—— xxx`、空 → `''`"做替换

`syncContent()` 会显式跳过 `[data-cap]` 和 `[data-byline]` 元素，避免主输入框的 `content` 被署名 div 误覆盖。

### 8.11 配色字段命名

配色面板和数据层都用了一组角色化命名（不再用过时的"主色 70% / 辅色 25%"）：

| 内部字段 | 面板 label | 用途 |
| --- | --- | --- |
| `text` | 字色 | 文字主色，也常用作大面积深色块底（如杂志深底 hero） |
| `bg` | 背景色 | 页面背景色 / 浅色块底 |
| `accent` | 点缀色 | 装饰线 / 标签 / 引用块底色等点缀 |
| `decor` | 装饰色 | 仅报纸用：暗红装饰文字色（卷期号、副标、署名）|

**JSON 占位符**：`{text}` / `{bg}` / `{accent}` / `{decor}`

**向后兼容**：旧版 JSON 字段名 `primary` / `secondary` / `accent2` 和占位符 `{primary}` / `{secondary}` / `{accent2}` 仍可工作，上传时自动按 `primary→text、secondary→bg、accent2→decor` 映射。新生成的模板请直接用新字段名。

## 10 已知限制

- 自定义模板没有持久化（刷新即丢）；如果 AI 生成的 JSON 里把颜色写死成 hex 而不是 `{text}/{bg}/{accent}` 占位符，配色面板调色对它无效
- 撤销（Ctrl+Z）作用于内容编辑、配色、层级切换等"可逆操作"；**不会撤销自定义模板的上传 / 删除**（文件操作走独立的 + / × 按钮）
- 图片：单张 ≤ 3MB、单块 ≤ 9 张、仅图片格式；上传后有时效，建议复制 HTML 后**立即发布**到公众号
- 框选 / 拖拽不支持触摸事件
- 不支持表格、视频、多列布局
- 手机端字体可能回退到系统默认（中文字体生态决定，公众号会保留 `font-family`，但安卓机普遍没装宋体 / 楷体）
