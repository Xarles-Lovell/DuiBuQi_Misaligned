# 对不齐 · Misaligned

> 对不起，你的排版对不齐！

一个为公众号原创推文设计的"一键排版"网页工具。左边写、右边实时预览，写完点一下复制就能直接粘贴进公众号后台。不需要懂 HTML，也不需要切图。

---

## 1 这是做什么的

写公众号文章时，最让人头疼的不是写内容，而是排版：什么字体、行距多少、图怎么放、标题怎么突出……一篇文章光排版就能耗掉一两个小时。

「对不齐」做的事情是：

- 你只管写内容，把它分成"标题 / 正文 / 图片 / 引用"这种简单的块
- 工具帮你按选好的风格自动排好版（杂志感 / 报纸感 / 便签感 / 你自定义的风格）
- 一键把成品 HTML 复制走，粘贴进公众号后台直接发

整个工具是一个**单文件 HTML**，不需要后端、不需要安装。

## 4 五分钟上手

1. **写内容** — 左侧直接点击文字修改。
2. **加块** — 编辑区底部一排「加块」按钮分三组：左边「引言」、中间一大组（一级 / 二级 / 三级标题、正文、引用、分隔线、图片）、右边「结尾」。新块插入到你**正在编辑的块下方**（没有焦点时插到末尾）。
3. **排序** — 鼠标移到块左侧，用 ⋮⋮ 手柄拖动到想要的位置。
4. **多选** — 块右侧的「✓」点亮就加入多选；或在编辑区空白处拖出方框框选。多选后任意一个选中块的 ×（删）、+（复制）、改层级、⋮⋮（拖）都会作用于整组。
5. **加图** — 点图片块，可一次选多张（最多 9 张）。下方图注框可以写一句说明。
6. **选风格** — 顶部切换"杂志 / 报纸 / 便签"，配色按钮里能改主色 / 辅色 / 点缀色。
7. **发布** — 等图全部上传完（图片块右下显示"✓ 全部已上传"），点右上角「复制 HTML」→ 打开公众号后台 → 新建图文 → 粘贴 → 完成。

> 上传的图片有有效期，复制 HTML 后请尽快粘贴进公众号发布。微信会把图片转存到自己的服务器，发布之后就不受影响了。

## 5 自定义模板

如果三种内置模板都不喜欢，可以让 AI 给你生成专属模板：

1. 顶部点「＋」按钮
2. 复制弹窗里的 Prompt（有"无参考图"和"有参考图"两个版本）粘贴给 Claude / ChatGPT 等
3. 把 AI 返回的 JSON 保存为 `.json` 文件
4. 上传 JSON，会出现一个新 tab，自动切到这个新模板

可以**一次上传多个**自定义模板，每个都会变成一个独立 tab，和内置模板并列。鼠标 hover tab 上会出现「×」，点掉就删除那个模板（如果删的是当前激活的，会自动回到「杂志」）。

**自定义模板也支持配色面板调色**：AI 生成的 JSON 里会带 `colors` 字段（这套风格的初始三色），HTML 里相应位置写成 `{primary}` `{secondary}` `{accent}` 占位符。上传后配色面板的预设按钮显示「自定义」，点它可以**恢复到该模板的初始配色**；调整任意颜色 → 预览实时变化；切到别的模板再切回来，颜色记忆保留。

**H1 自动编号占位符**：每个 H1 会被自动编号，渲染时替换 JSON 里的占位符：

| 占位符 | 渲染示例（第 3 个 H1 时） |
| --- | --- |
| `{number}` | `3` |
| `{numberPad2}` | `03` |
| `{numberEng}` | `THREE` |
| `{numberCn}` | `叁` |

`{numberEng}` 和 `{numberCn}` 支持到 99（`NINETY-NINE` / `玖拾玖`），超出回落到阿拉伯数字。AI 生成 JSON 时建议挑一个跟风格搭的占位符。

> ⚠ 自定义模板只存在浏览器会话里，**刷新或关闭页面后会全部消失**。把 JSON 文件保存好，下次再用重新上传。

## 6 目录结构

```
Misaligned/
├── index.html       # 单文件应用，所有逻辑、样式都在里面
├── README.md        # 当前文件
└── *.json           # 用户上传过的自定义模板源文件（可有可无，仅作备份）
```

只有一个 HTML 文件是必须的，没有依赖、没有构建。`*.json` 是用户用 AI 生成的模板源文件，用完不删也行（页面不会自动加载，每次都要手动上传）。

## 8 技术说明

- 纯 HTML + CSS + 原生 JS，**零依赖**
- 无打包步骤，源文件即产物
- 用到的外部资源：
  - Google Fonts CDN：`Playfair Display`（英文 logo 字体）
  - 第三方图片托管服务：用于把本地图片转成可被公众号抓取的 URL
- 富文本复制使用 `navigator.clipboard.write()` + `ClipboardItem` 写入 `text/html`，降级方案是 `contenteditable + execCommand('copy')`

### 8.0 公众号兼容性红线（改模板前必读）

公众号编辑器粘贴 HTML 时会做"标签白名单过滤"：

- **会被剥掉的**：`<div>` —— 标签连同上面的内联 `style` **整个被丢弃**，色块、卡片、彩色背景全没了
- **保留的**：`<section>` / `<p>` / `<span>` / `<table>` / `<h2>` / `<h3>` / `<img>` 等

所以三个内置模板（`tmpl1` / `tmpl2` / `tmpl3`）以及通用图片渲染（`renderImage`）**所有色块和容器都用 `<section>` 而不是 `<div>`**。改模板时务必继续遵守。预览侧浏览器对 `<section>` 和 `<div>` 渲染表现完全一致，不会有视觉差异。

> 自定义模板的 AI 生成 Prompt 也已在要求里强调"容器必须用 `<section>`"。如果用户的 JSON 里出现 `<div>` 容器，复制到公众号后会丢样式——这是已知限制，不是 bug。

### 8.1 统一排版规范（所有模板必须遵循）

为了让杂志 / 报纸 / 便签 / 自定义任何模板切换时视觉感受一致，统一一套**字号 / 加粗 / 行高 / 缩进 / 间距**规范。代码里集中定义在 `TYPO` 常量，AI 生成自定义模板的 Prompt 也写死了同一套。

| 块类型 | 字号 | 加粗 | 行高 | 备注 |
| --- | --- | --- | --- | --- |
| h1 | 24px | ✓ | 1.5 | 文章一级标题（用户在公众号输入框写的"主标题"是另一回事） |
| h2 | 18px | ✓ | 1.5 | |
| h3 | 16px | ✓ | 1.5 | |
| body | 16px | ✗ | 1.9 | **首行缩进 2em** |
| quote | 15px | ✗ | 1.85 | italic |
| cover（引言） | 15px | ✗ | 1.85 | italic、淡灰色 |
| footer | 14px | ✗ | 1.8 | |

- **左右内边距**：22px（wrapper 自带或每块自带，最终视觉一致）
- **加粗只用在 h1/h2/h3**，其他块**默认不加粗**；用户可以在编辑器里用「B」按钮或 Ctrl+B 自己手动加粗某段文字
- **首行缩进只对 body**

新增内置模板或修改现有模板时，**字号 / 加粗 / 缩进必须从 `TYPO` 取值**，不要硬编码。

#### 风格特例（允许豁免）

某些块在特定风格下是这套风格的"招牌"，刻意**不遵循 TYPO 规范**。豁免要在代码里加注释解释原因：

| 模板 | 块 | 豁免内容 | 原因 |
| --- | --- | --- | --- |
| 报纸 | cover | 30px / weight 900 / letter-spacing 2px / 不 italic / 通栏深底色 | 模仿真实报纸头版，是这套风格的灵魂 |
| 报纸 | quote | 15px italic + 大引号 + 署名 + 米色双线 | 模仿报纸社论引文 |
| 便签 | quote | 底色用纯 `accent`（不透明）而非半透明叠加 | 让用户调色时引用块底色直接对应点缀色，调色无障碍 |

新增风格特例时也要在 README 这张表里登记一行，避免后人误以为是 bug 改回去。

#### H1 自动编号

每个 H1 块会被自动编号，渲染时三个内置模板和自定义模板都会用到：

| 模板 | 装饰文字 | 编号格式 |
| --- | --- | --- |
| 杂志 | `NUMBER 01` | 两位数 padding |
| 报纸 | `NUMBER · ONE` | 英文单词（1–99） |
| 便签 | `其 壹` | 中文大写（1–99） |

代码里实现在 `numPad2()` / `numEng()` / `numCnUpper()` 三个工具函数。自定义模板可在 JSON 的 h1 里用 `{number}` / `{numberPad2}` / `{numberEng}` / `{numberCn}` 占位符引用对应格式。

### 8.2 图片托管 API Key 的安全性（给开发者看）

> 这一节面向打算 fork 这个项目的人，普通使用者跳过即可。

当前实现里图片托管 API Key 直接写在前端源码里。**对于纯静态部署（如 GitHub Pages），任何前端调用第三方 API 的 Key 都会被浏览器加载到，没有办法靠"环境变量"或"混淆"真正隐藏**——只要打开浏览器调试工具就能看到。

如果你 fork 了这个项目并想把自己的 Key 真正藏起来，标准做法是搭一个**反向代理**。免费方案推荐 [Cloudflare Workers](https://workers.cloudflare.com/)：

1. 在 Cloudflare 注册一个 Worker，把图片托管 Key 写在 Worker 的环境变量里
2. Worker 接收前端 POST 的图片 → 用环境变量里的 Key 调上游 API → 把上游响应转发给前端
3. 前端的 fetch 地址改成你的 Worker URL，源码里完全不出现 Key

这样：
- Key 永不进入浏览器
- 还能在 Worker 层加访问限制（来源域名白名单、速率限制）防止被滥用

需要详细的 Worker 代码示例可以提 issue。

### 8.3 关于"上传后立即删除"

理论上希望"用户在前端删了图，图床那边也立即删除"。但当前使用的免费图片托管服务**没有提供可程序化调用的删除接口**（其响应里的 `delete_url` 是给人手动点的网页），因此暂时只能依赖**自动过期机制**——上传时设置较短的过期时间，到期自动清除。

如果改用其他支持 API 删除的服务（如自建 MinIO、阿里云 OSS、Cloudinary 等），可以实现"删除即删"。

### 8.4 按钮主 / 副样式规范

| 类型 | 视觉 | 用法 | 当前实例 |
| --- | --- | --- | --- |
| 主按钮 | 黑底白字（`.tbtn.primary` / `.tbarbtn.primary`） | "关键操作"或"当前最重要的下一步" | 复制 HTML、使用手册、导入 Markdown、撤销（在有可撤销内容时动态切换） |
| 副按钮 | 白底深字（`.tbtn` / `.tbarbtn`） | 一般操作 | 配色、重做、高亮 / 取消高亮、纸张切换 |

新增按钮时按这套挑 class，不要随便发明第三种颜色。"撤销"是动态例子：`refreshUndoRedoUI()` 在 `historyStack.length > 0` 时给它加 `primary` 类。

### 8.5 图片渲染规则

图片块的渲染分两端，逻辑不同：

| 端 | 单图 | 多图 |
| --- | --- | --- |
| **左侧编辑器**（缩略图） | 120×120，`object-fit:cover` 1:1 裁剪 | 120×120 一组 flex 自动换行，相同尺寸 |
| **右侧预览 / 复制 HTML** | 撑满容器宽度，原比例 | 每张固定宽度（默认 150px）+ `inline-block` + `text-align:center`：自动居中、自动换行、保持原比例不裁剪 |

设计取舍：左侧编辑器追求"等大缩略整齐"方便管理；右侧追求"忠于原图"避免裁掉重要内容。改动 `renderImage()` 时记得这条不要混淆。

#### 上传校验

- 单张 ≤ **3MB**（受 imgBB 免费额度限制；超出就提示用户压缩）
- 仅 JPG / PNG / WebP / GIF / BMP（白名单，其他类型给具体格式名提示）
- 单块 ≤ **9 张**（满了再传会拒绝并提示"先删除几张"）

三类错误都用 `toast()` 给出**可执行的下一步**（不是抽象的"上传失败"）。

## 10 已知限制

- 自定义模板没有持久化（刷新即丢）；如果 AI 生成的 JSON 里把颜色写死成 hex 而不是 `{primary}/{secondary}/{accent}` 占位符，配色面板调色对它无效
- 撤销（Ctrl+Z）作用于内容编辑、配色、层级切换等"可逆操作"；**不会撤销自定义模板的上传 / 删除**（文件操作走独立的 + / × 按钮）
- 图片：单张 ≤ 3MB、单块 ≤ 9 张、仅图片格式；上传后有时效，建议复制 HTML 后**立即发布**到公众号
- 框选 / 拖拽不支持触摸事件
- 不支持表格、视频、多列布局
